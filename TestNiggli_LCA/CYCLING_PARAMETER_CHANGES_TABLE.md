## Analysis

### Key Observations

1. **g1 and g2 are absolutely constant**
   - No changes at all (Δ = 0 for all 20 iterations)
   - These are the fundamental lattice parameters that define the reduced cell

2. **g3 oscillates at numerical noise level**
   - Changes of ±1.42×10⁻¹⁴ (log₁₀ ≈ -13.85)
   - This is at machine precision (IEEE 754 double precision ≈ 10⁻¹⁵)
   - Effectively constant for practical purposes

3. **trace is constant**
   - Changes of ±1.42×10⁻¹⁴ (same as g3)
   - Proves no net reduction is occurring
   - Algorithm is cycling without progress

4. **g4, g5, g6 show large oscillations**
   - Changes of order 10² (log₁₀ ≈ 2)
   - These are the scalar products (related to angles)
   - Cycling between three distinct states

5. **Perfect 3-iteration periodicity**
   - State A → State B → State C → State A (repeats)
   - Each state differs significantly in g4, g5, g6
   - But all have identical trace (g1+g2+g3)

### Why the Algorithm Can't Detect Cycling

**The step conditions only check:**
- Relationships between parameters (e.g., |g4| > g2)
- Sign patterns
- Tie-breaker inequalities

**The step conditions do NOT check:**
- Whether trace is changing
- Whether we've been in this state before
- How many iterations have passed without progress

**Result:** Steps keep firing because their conditions are satisfied, even though no net reduction is occurring.
